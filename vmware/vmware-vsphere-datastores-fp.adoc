= VMware 데이터 저장소용 FabricPool
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font

[role="lead"]
가상 머신 스토리지는 게스트 OS 및 하이퍼바이저 계층에서 ONTAP로 흐르며, 여기서 FabricPool 계층화 및 WAFL을 통해 성능, 효율성 및 비용 최적화를 제공합니다.

ONTAP 9.6 릴리스부터 FabricPool는 VMware 데이터스토어를 포함한 가상화된 워크로드를 지원합니다. 이제 VMware 데이터스토어를 지원하는 FlexVol 볼륨(VM 템플릿, 전원이 꺼진 VM, 스냅샷, 백업 등)에서 콜드 블록을 객체 스토리지로 계층화하고 핫 데이터는 로컬에 유지할 수 있습니다.

== 컴퓨팅 및 가상 디스크 계층

게스트 운영 체제는 VMDK, QCOW2 또는 VHDX와 같은 가상 디스크 형식에서 실행되며, 이러한 형식은 애플리케이션과 사용자가 액세스할 수 있는 컴퓨팅 및 블록 장치 계층을 생성합니다.

게스트 운영 체제는 가상 디스크를 블록 장치로 간주한다는 점에 유의해야 합니다. 따라서 무중단 업그레이드, 하드웨어 오류 또는 기타 예기치 않은 이벤트 발생 시 내결함성을 계획할 때 게스트 내 블록 장치 시간 초과를 고려해야 합니다.


TIP: link:https://www.netapp.com/pdf.html?item=/media/17239-tr-4598.pdf["FabricPool 모범 사례"]에 따라 게스트 OS와 관련된 가상 디스크 파일의 블록 특성으로 인해 NAS 프로토콜을 사용하더라도 가상 머신 데이터스토어를 호스팅하는 FlexVol 볼륨은 SAN 스토리지로 취급해야 합니다. 따라서 NetApp에서는 스냅샷 전용 티어링 정책을 사용할 것을 권장합니다. 중요하지 않은 데이터에는 자동 정책을 사용할 수 있지만, 클라우드 티어와의 연결이 끊어지면 시스템이 충돌하거나 읽기 전용 상태로 전환될 수 있으므로 주의해야 합니다.



== 하이퍼바이저 역할

하이퍼바이저는 실행 중인 가상 머신과 CPU, 메모리, 스토리지와 같은 물리적 리소스 간의 연결을 중재합니다. NAS 사용 사례에서 하이퍼바이저는 NAS 공유를 마운트하고 가상 디스크 파일을 블록 장치로 가상 머신에 제공합니다.

하이퍼바이저 파일 시스템 경로는 VMFS, NTFS, EXT4 등과 같은 하이퍼바이저별 파일 시스템으로 포맷된 iSCSI/FCP로 연결된 LUN 또는 NVMe 네임스페이스를 통해 VM 스토리지를 포함합니다.

NAS 파일 시스템 경로는 NFS 또는 SMB 프로토콜을 사용하여 가상 머신 스토리지용 ONTAP 볼륨에 대한 직접적인 파일 레벨 액세스를 제공합니다.

두 경로는 ONTAP의 WAFL 계층에서 수렴하여 스냅샷 및 메타데이터 관리와 같은 일관된 데이터 서비스를 가능하게 합니다.



== FabricPool 계층화 통합

FabricPool은 고성능 로컬 스토리지와 저비용 클라우드 객체 계층 간의 데이터 배치를 지능적으로 관리합니다.

FabricPool는 데이터 세트에 적용된 FabricPool 계층화 정책에 따라 로컬 고성능 스토리지와 확장 가능한 클라우드 객체 스토리지 간에 데이터를 자동으로 계층화합니다.

ONTAP의 FabricPool 통합을 통해 로컬 읽기 성능 저하 없이 하이브리드 클라우드 환경 전반에 걸쳐 효율적인 데이터 배치가 가능합니다.



== 스토리지 효율성 및 관리

FabricPool를 활성화하기 전에 작동 방식을 이해하는 것이 중요합니다. 올바른 구성은 적절한 데이터를 적절한 계층에 저장함으로써 성능을 향상시키고 위험을 줄이며 총소유비용(TCO)을 최적화하는 데 도움이 됩니다.

SAN/NVMe와 NAS 파일 프로토콜 경로는 모두 WAFL 계층에서 수렴합니다. WAFL 파일 시스템은 메타데이터 일관성을 보장하고, 스냅샷 및 클론 생성을 지원하며, SSD, HDD 및 오브젝트 스토리지 매체에 대한 쓰기를 최적화합니다.
