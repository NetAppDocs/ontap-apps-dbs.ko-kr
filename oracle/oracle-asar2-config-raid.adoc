= RAID
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
RAID는 드라이브 고장으로부터 데이터를 보호하기 위해 패리티 기반 중복성을 사용하는 것을 의미합니다. ASA r2는 AFF 및 FAS 시스템과 동일한 ONTAP RAID 기술을 사용하여 여러 디스크 오류에 대한 강력한 보호 기능을 제공합니다.

ONTAP ASA r2 시스템에 대한 RAID 구성을 자동으로 수행합니다. 이는 ASA r2의 핵심 기능 중 하나로, 간소화된 스토리지 관리 환경을 제공합니다.

ASA r2의 자동 RAID 구성과 관련된 주요 세부 정보는 다음과 같습니다.

* 스토리지 가용성 영역(SAZ): 기존의 집계 및 RAID 그룹을 수동으로 관리하는 대신, ASA r2는 스토리지 가용성 영역(SAZ)을 사용합니다. 이는 고가용성(HA) 쌍을 위한 공유 RAID 보호 디스크 풀로, 두 노드 모두 동일한 스토리지에 대한 완전한 액세스 권한을 갖습니다.
* 자동 배치: 스토리지 장치(LUN 또는 NVMe 네임스페이스)가 생성될 때 ONTAP SAZ 내에 볼륨을 자동으로 생성하고 최적의 성능과 용량 균형을 위해 배치합니다.
* 수동 집계 관리 불가: ASA r2에서는 기존의 집계 및 RAID 그룹 관리 명령이 지원되지 않습니다. 이를 통해 관리자는 RAID 그룹 크기, 패리티 디스크 또는 노드 할당을 수동으로 계획할 필요가 없어집니다.
* 간소화된 프로비저닝: 프로비저닝은 시스템 관리자 또는 간소화된 CLI 명령을 통해 처리되며, 기본 물리적 RAID 구성보다는 스토리지 장치에 초점을 맞춥니다.
* 워크로드 재분배: 2025 릴리스(ONTAP 9.17.1)부터 ONTAP 수동 개입 없이 성능과 공간 활용도의 균형을 유지하기 위해 HA 쌍의 노드 간 워크로드를 자동으로 재분배합니다.


ASA r2는 대부분의 구성에서 RAID DP , 매우 큰 SSD 풀에서는 RAID-TEC 사용하는 ONTAP의 기본 RAID 기술을 자동으로 사용합니다. 이로써 RAID를 수동으로 선택할 필요가 없어집니다. 이러한 패리티 기반 RAID 레벨은 미러링보다 스토리지 효율성과 안정성이 뛰어납니다. 미러링은 이전 Oracle 모범 사례에서 종종 권장되지만 ASA r2에는 적용되지 않습니다. ONTAP WAFL 통합을 통해 기존 RAID의 쓰기 성능 저하 문제를 해결하여 리두 로깅 및 임의 데이터 파일 쓰기와 같은 Oracle 워크로드에 최적의 성능을 보장합니다. ASA r2는 자동화된 RAID 관리 및 스토리지 가용성 영역과 결합하여 Oracle 데이터베이스에 대한 고가용성과 엔터프라이즈급 보호 기능을 제공합니다.
