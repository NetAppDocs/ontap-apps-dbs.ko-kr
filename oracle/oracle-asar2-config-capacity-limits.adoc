= 용량 관리
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
예측 가능하고 관리가 용이한 고성능 엔터프라이즈 스토리지로 데이터베이스나 기타 엔터프라이즈 애플리케이션을 관리하려면 데이터 및 메타데이터 관리를 위해 드라이브에 여유 공간이 필요합니다. 필요한 여유 공간의 양은 사용하는 드라이브의 유형과 비즈니스 프로세스에 따라 다릅니다.

ASA r2는 애그리게이트 대신 스토리지 가용 영역(SAZ)을 사용하지만 원칙은 동일합니다. 즉, 여유 공간에는 실제 데이터, 스냅샷 또는 시스템 오버헤드로 인해 소비되지 않은 모든 물리적 용량이 포함됩니다. 씬 프로비저닝 또한 고려해야 할 사항입니다. 논리적 할당은 실제 물리적 사용량을 반영하지 않습니다.

엔터프라이즈 애플리케이션에 사용되는 ASA r2 스토리지 시스템에 대한 NetApp 권장 사항은 다음과 같습니다.



== ASA r2 시스템의 SSD 풀


TIP: NetApp ASA r2 환경에서 최소 10%의 여유 물리적 공간을 유지할 것을 권장합니다. 이 지침은 ASA r2 시스템에서 사용하는 SSD 전용 풀에 적용되며 SAZ 및 스토리지 장치 내의 사용되지 않은 모든 공간을 포함합니다. 논리적 공간은 중요하지 않습니다. 중요한 것은 데이터를 저장하는 데 실제로 사용할 수 있는 물리적 공간입니다.

ASA r2는 성능 저하 없이 높은 사용률을 유지할 수 있지만, 거의 최대 용량으로 작동할 경우 저장 공간 부족 위험이 증가하고 스토리지 확장 시 관리 부담이 커집니다. 사용률이 90%를 넘더라도 성능에는 영향을 미치지 않을 수 있지만, 관리가 복잡해지고 추가 드라이브 프로비저닝이 지연될 수 있습니다.

ASA r2 시스템은 최대 128TB의 스토리지 장치와 HA 쌍당 최대 2PB의 SAZ 크기를 지원하며, ONTAP 노드 간 용량 균형을 자동으로 조정합니다. 클러스터, SAZ 및 스토리지 장치 수준에서 사용률을 모니터링하는 것은 스냅샷, 씬 프로비저닝 워크로드 및 향후 성장을 위한 충분한 여유 공간을 확보하는 데 필수적입니다. 용량이 임계점(사용률 약 90%)에 근접하면 성능과 복원력을 유지하기 위해 SSD를 그룹(최소 6개)으로 추가해야 합니다.
