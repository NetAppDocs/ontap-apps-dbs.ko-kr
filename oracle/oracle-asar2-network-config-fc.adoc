= FC SAN 구성
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
ASA r2 시스템에서 Oracle 데이터베이스용 FC SAN을 구성하는 것은 기본적으로 표준 SAN 모범 사례를 따르는 것입니다.

ASA r2는 SAN 전용 워크로드에 최적화되어 있으므로 성능, 복원력 및 단순성에 중점을 둔 AFF/ FAS 와 동일한 원칙을 따릅니다. 여기에는 호스트와 스토리지 시스템 간의 SAN에 충분한 대역폭이 확보되어 있는지 확인하고, 필요한 모든 장치 간에 모든 SAN 경로가 존재하는지 확인하고, FC 스위치 공급업체에서 요구하는 FC 포트 설정을 사용하고, ISL 경합을 방지하고, 적절한 SAN 패브릭 모니터링을 사용하는 것과 같은 일반적인 계획 조치가 포함됩니다.



== 조닝

FC 존은 하나 이상의 이니시에이터를 포함할 수 없습니다. 이 방식은 처음에는 효과가 있는 것처럼 보이지만 이니시에이터 간 혼선이 결국에는 성능과 안정성을 저하시킵니다.

여러 공급업체의 FC 타겟 포트 동작이 드물게 문제를 일으키기도 하지만 멀티타겟 존은 일반적으로 안전하다고 간주됩니다. 예를 들어, NetApp와 NetApp이 아닌 스토리지 어레이의 타겟 포트를 같은 존에 포함하지 마십시오. 또한, NetApp 스토리지 시스템과 테이프 장치를 같은 존에 배치하면 문제가 발생할 가능성이 더 높아집니다.

[NOTE]
====
* ASA r2는 애그리게이트 대신 스토리지 가용 영역을 사용하지만, 이는 FC 영역 설정 원칙을 변경하지는 않습니다.
* 멀티패싱(MPIO)은 여전히 ​​주요 복원력 메커니즘입니다. 그러나 대칭형 액티브-액티브 멀티패싱을 지원하는 ASA r2 시스템의 경우 LUN에 대한 모든 경로가 활성화되어 I/O에 동시에 사용됩니다.
