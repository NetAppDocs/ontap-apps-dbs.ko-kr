= 정합성 보장 그룹 백업
:allow-uri-read: 


[role="lead"]
가장 간단한 백업 방법은 전체 Oracle 데이터베이스를 단일 ONTAP 볼륨, LUN 또는 NVMe 네임스페이스에 저장하는 것입니다.

데이터 파일, 아카이브 로그, 리두 로그 및 제어 파일을 단일 스냅샷으로 보호하는 것은 유효한 백업, 복원 및 복제 방법입니다. 그러나 RPO는 백업 시점으로 제한됩니다. 이 방법은 RPO가 1시간 이상인 경우에 적합합니다. 데이터베이스가 AFF 볼륨 또는 ASA 스토리지 장치에 걸쳐 있는 경우, 앞서 설명한 도구 중 하나를 사용하여 일관성 그룹을 생성해야 합니다.

예를 들어, 전체 데이터베이스를 다음과 같은 스냅샷 일정으로 단일 AFF 볼륨에 저장할 수 있습니다.

* 시간별 스냅샷 72개
* 30개의 야간 스냅샷
* 12개의 월별 스냅샷


이 기능은 이전 72시간 동안의 누적 백업을 기준으로 1시간의 RPO(복구 목표 시간)를 제공하며, 추가로 야간 및 월간 백업을 수행합니다. 또한 여러 데이터베이스 또는 애플리케이션 파일을 단일 볼륨 또는 일관성 그룹 스냅샷 세트에 포함하여 더 큰 환경에서 일관된 백업을 제공할 수 있습니다.
